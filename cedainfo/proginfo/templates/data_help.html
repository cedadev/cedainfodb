Data products are the component parts of a data set. They are consistent, and coherent. The will generally come form one data production tool or a consistant group of tools.

Below are the attributes needed for data products and what they mean.
Attribute nameMeaningValue in for this data product
DPT A name for the data production tool. {{ data.DPT }}
Numobs Number of observation sites or platforms. Multipule sites means more metadata records. {{ data.Numobs }}
Numact Number of activities assosiated with the DPT. This could be number of campaigns. Multipule activites means more metadata records. {{ data.Numact }}
Datavol Expected final data volume in TB. Small volumes should be listed as 0. If the dataset is going to grow for an indefinit period then the estimate should be for the 5 years worth of data. {{ data.Numact }}
Contact1 A data contact person, often the name of the person producing the data. {{ data.Contact1 }}
Contact2 A secondary data contact person. {{ data.Contact2 }}
Deliverydate The expected date of last data arrival at the archive. {{ data.Deliverydate }}
Formats A short note on the expected formats of the data. {{ data.formats }}
IPR A short note on interlectual property right issues for this data. {{ data.ipr }}
Versions A short note on versioning issues for this data. {{ data.versions }}
Numparameters Number of expected parameters. {{ data.Numparameters }}
Preservation The intended preservation. For example, "keep for 5 year after project ends" or "keep indefinately". {{ data.Preservation }}
Unresponsive If the data contacts are unresponsive then this flag should be ticked. {{ data.Unresponsive }}
Proj This is the link to the project generating the data. {{ data.proj }}
Status The current state of the data product.
  • Undefined: Data products have a loose or indeterminate description.
  • Defined: Data products have firm parameters ? known size, feature type, DPT, etc.
  • In production: Data products are being produced by the project, but none has yet been delivered.
  • Delivery: Data products are being delivered, but are not yet complete.
  • Archived: The data product is complete and not more is expected.
{{ data.status }}
Notes Add notes about the data products by typing text in the box and clicking "Add note". Text starting with http:// will appear as links, text starting with Q followed by a number will be a displayed as a link to a query (e.g. Q17), and text starting with a # then a number (e.g. #5000) links to a CEDA ticket.
