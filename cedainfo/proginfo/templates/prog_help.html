A programme is a data producing activity that has a coherent funding line for data management. Examples are URGENT (a directed mode programme), NCAS-Weather (an NCAS core activitiy) or consortium grants (a NERC programme). It's a programme if this is the level at which the data management funding is administered.

The tool here is desighed to record a minimum information to scope and track the programme. Below are the attributes needed and what they mean.
Attribute nameMeaningValue in for this programme
Name The short name of the programme. {{ prog.name }}
Title The full title or long name of the programme. {{ prog.title }}
Description A short description of the programme. This can come from the NERC website. {{ prog.description|truncatewords:30 }}
SciSupContact The CEDA person responsible for dealing with the data mangment for this programme {{ prog.sciSupContact }}
FundingContact The NERC or other funding body contact. This is person we contact for matters to do with budgets. {{ prog.FundingContact }}
ScienceContact Our main programme contact. This could be a science coordinator or a steering committee chair. {{ prog.ScienceContact }}
ProgrammeSize The financal size of the programme in k. This gives us an idea of the scale and how many people we need to talk to. {{ prog.ProgrammeSize }}
Projectcode This is the STFC project code that relates to activities for this programme. Some programmes will have many codes and these may be recorded at the project level as notes. {{ prog.projectcode }}
ScopingFunds Large projects may have funds specifically for scoping. This should be in k. {{ prog.ScopingFunds }}
StartDate The start date of the programme. {{ prog.StartDate }}
EndDate The end date of the programme. {{ prog.EndDate }}
State The current state of the programme.
  • Unassessed: The programme is known, but no assessment has been carried out.
  • Assessed: The programme has been assessed with all the relevant info found, and found the programme is in scope for our data centres. All projects in the programme have a state of "Not scoped".
  • Out of scope: The programme is not in the scope of the data centres. Not action required by the data centres.
  • Scoping: The projects in the programme are being scooped. The initial DMP is being created. (There are some projects in the scoping phase and the DMP has not been accepted).
  • Traking: The DMP has been accepted by the SC. Any projects in an accepted DMP are moved to an "active state". Adding more projects that need scoping for a new round or late reporting does not move the programme state back to scoping, thus it is possible to have scoping projects in a DMP approved programme.
  • Complete: All projects are complete.
 {{ prog.state }}
Progtype The type of programme. This is uncontrolled and is there to flag similar programmes e.g. directed mode. {{ prog.progtype }}
Unresponsive If the science and funding contacts are unresponsive. Then this flag should be ticked. {{ prog.Unresponsive }}
Projects This is the list of assosiated projects. To add another project click on "Add new project". To edit an existing project click on items from the list.
Notes Add notes about the programme by typing text in the box and clicking "Add note". Text starting with http:// will appear as links, text starting with Q followed by a number will be a displayed as a link to a query (e.g. Q17), and text starting with a # then a number (e.g. #5000) links to a CEDA ticket.
